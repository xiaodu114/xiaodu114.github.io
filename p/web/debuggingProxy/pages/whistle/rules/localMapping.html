本地映射

本地映射的核心逻辑是建立一张映射表：远程 URL <-> 本地文件路径。当代理拦截到远程 URL 时，直接去查这张表，返回对应的本地文件。

解释

响应体替换

站在 HTTP 协议和代理引擎的角度，它做了一件非常纯粹的事情：Response Body Replacement。

  • 代理服务器并没有真的去请求目标服务器，而是伪造了一个响应。它读取本地文件的内容，作为 HTTP 响应的 Body 塞进去，然后连同状态码（200）一起返回给客户端。
  • 在 whistle 的内部架构中，它本质上就是一个响应拦截处理器。

远程资源本地化 / 线上转本地

在前端具体的业务场景中，我们通常用这类词汇来描述它的目的。

  • 线上转本地： 线上出了 Bug，但不想重新走一遍发布流程，直接把线上的请求“转”到本地代码。
  • 动静分离代理： 如果只 Map Local 静态资源（JS/CSS/图片），而让接口继续走线上，这种策略常被称为动静分离调试。
